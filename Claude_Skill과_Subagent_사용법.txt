======================================================================
  Claude Code - Skill과 Subagent 쉬운 사용 가이드 (한국여행 지도 프로젝트용)
======================================================================

📌 개요
----------------------------------------------------------------------
이 문서는 현재 "국내여행" 폴더의 한국여행 지도 프로젝트에서 사용할 수 있는
Skill과 Subagent의 실전 활용법을 설명합니다.

**현재 프로젝트 구조**:
국내여행/
├── skills/                          # 프로젝트 전용 Skill
│   ├── korea-map-init.yaml         # 프로젝트 초기화 Skill
│   └── data-collector.yaml         # 데이터 수집 Skill
├── 부산여행_브라운스위트부산역_맛집_관광지.csv
├── KOREA_TRAVEL_MAP.md
└── KOREA_MAP_SKILLS.md

Claude Code에는 작업을 더 효율적으로 처리할 수 있는 두 가지 강력한 도구가 있습니다:
1. Skill (스킬) - 미리 만들어진 전문 명령어 (이 프로젝트 전용 포함)
2. Subagent (서브에이전트) - 복잡한 작업을 자동으로 처리하는 AI 보조원


======================================================================
1️⃣ SKILL (스킬) - 전문 명령어 시스템
======================================================================

🔹 Skill이란?
----------------------------------------------------------------------
- 슬래시 커맨드(/)로 시작하는 전문화된 명령어
- 특정 작업을 빠르고 효율적으로 처리하도록 설계됨
- 예: /commit, /review-pr, /test, /document 등


🔹 이 프로젝트에서 사용 가능한 Skill
----------------------------------------------------------------------

🗺️ **한국여행 지도 프로젝트 전용 Skill** (skills/ 폴더):
  • /korea-map-init     - 한국여행 지도 프로젝트 초기화
                          (디렉토리 구조, HTML/CSS/JS 생성, 샘플 데이터)

  • /data-collector     - 관광지/맛집/숙소 데이터 자동 수집
                          (웹 크롤링, JSON 생성, 좌표 변환, 이미지 다운로드)

📂 **일반 개발 Skill** (SuperClaude 기본 제공):
  • /build          - 프로젝트 빌드 및 컴파일
  • /implement      - 기능 구현 (컴포넌트, API, 서비스 등)
  • /design         - 시스템 설계 및 아키텍처

📊 **분석 관련**:
  • /analyze        - 코드 및 시스템 분석
  • /troubleshoot   - 문제 조사 및 해결
  • /explain        - 코드 설명 및 교육

✨ **품질 관련**:
  • /improve        - 코드 개선 및 최적화
  • /cleanup        - 코드 정리 및 기술 부채 제거

🧪 **테스트 관련**:
  • /test           - 테스트 실행 및 리포트 생성

📝 **문서 관련**:
  • /document       - 문서 자동 생성
  • /git            - Git 작업 지원 (커밋 메시지 등)

📋 **계획 관련**:
  • /workflow       - 구현 워크플로우 생성
  • /estimate       - 개발 일정 추정
  • /task           - 장기 프로젝트 관리

🔧 **메타 관련**:
  • /index          - 프로젝트 문서화 및 지식 베이스
  • /load           - 프로젝트 컨텍스트 로딩
  • /spawn          - 복잡한 작업 분할 및 실행


🔹 Skill 사용 방법
----------------------------------------------------------------------

기본 사용법:
  /[skill-name]

인자와 함께 사용:
  /[skill-name] [arguments] [flags]

💡 **이 프로젝트에서의 실제 사용 예시**:

📍 한국여행 지도 프로젝트 전용 Skill:
  /korea-map-init --region seoul --days 3
  → 서울 3일 여행 프로젝트 초기화 (디렉토리, HTML/JS/CSS 자동 생성)

  /korea-map-init --region jeju --theme korean-traditional --days 5
  → 제주도 전통 테마 5일 프로젝트 생성

  /data-collector --type attractions --region busan --count 20
  → 부산 관광지 20개 데이터 자동 수집 및 JSON 생성

  /data-collector --type restaurants --keyword "부산 해운대 맛집" --auto-geocode
  → 키워드로 맛집 검색, 주소→좌표 자동 변환

  /data-collector --type hotels --region jeju --download-images
  → 제주 숙소 수집, 이미지 자동 다운로드 및 최적화

📍 일반 Skill 활용:
  /commit                              # Git 커밋 생성
  /analyze --focus performance         # 웹 성능 분석
  /document @data/                     # data 폴더 문서화
  /improve --delegate files            # 파일별 코드 개선 (Subagent 자동 활성화)


🔹 언제 Skill을 사용하나?
----------------------------------------------------------------------
✅ 반복적인 작업을 빠르게 처리할 때
✅ 특정 도메인(Git, 테스트, 문서 등)의 전문 작업이 필요할 때
✅ 일관된 품질의 결과물이 필요할 때
✅ 복잡한 설정 없이 바로 시작하고 싶을 때


======================================================================
2️⃣ SUBAGENT (서브에이전트) - AI 보조 작업자
======================================================================

🔹 Subagent란?
----------------------------------------------------------------------
- 복잡한 작업을 자율적으로 처리하는 전문 AI 에이전트
- 여러 단계의 작업을 독립적으로 수행
- 메인 Claude와 병렬로 작업하여 효율성 향상
- 이 프로젝트에서는 데이터 수집, 파일 분석, 품질 검증 등에 자동으로 활용됨


🔹 주요 Subagent 종류
----------------------------------------------------------------------

🔧 **Bash Agent**:
  - 용도: Git 작업, 명령 실행, 터미널 작업
  - 도구: Bash 명령어
  - 이 프로젝트 예시: 이미지 다운로드, 파일 압축, 서버 실행

🔍 **Explore Agent**:
  - 용도: 코드베이스 탐색 및 검색
  - 속도: 빠른 파일 및 키워드 검색
  - 레벨: quick, medium, very thorough
  - 이 프로젝트 예시: 데이터 파일 탐색, 중복 검색

🎯 **General-Purpose Agent**:
  - 용도: 복잡한 질문 연구, 다단계 작업
  - 도구: 모든 도구 사용 가능
  - 이 프로젝트 예시: 웹 크롤링, API 호출, 데이터 정제

📐 **Plan Agent**:
  - 용도: 구현 전략 설계
  - 출력: 단계별 계획, 중요 파일 식별, 아키텍처 고려사항
  - 이 프로젝트 예시: 새 기능 추가 계획, 구조 개선 전략

🗺️ **프로젝트 전용 Subagent** (자동 활성화):
  - **GeoJSON Expert**: 좌표 변환, 거리 계산, 위치 데이터 처리
  - **Leaflet Specialist**: 지도 마커, 클러스터링, 타일 레이어 관리
  - **Travel Data Curator**: 데이터 검증, 중복 제거, 품질 관리
  - **UI Designer**: 한국 전통색 적용, 타이포그래피 최적화
  - **Performance Tuner**: 이미지 최적화, 번들 압축, 렌더링 개선


🔹 Subagent 사용 시나리오
----------------------------------------------------------------------

자동 활성화 조건:
  ✓ 파일 개수 > 50개
  ✓ 디렉토리 개수 > 7개
  ✓ 복잡도 > 0.8
  ✓ 멀티 도메인 작업 (3개 이상 영역)

수동 활성화:
  --delegate [files|folders|auto]
  --concurrency [1-15]


🔹 Subagent 작동 방식
----------------------------------------------------------------------

1. 작업 분석
   └─> 복잡도 평가 → 병렬 처리 가능 여부 확인

2. Subagent 할당
   └─> 전문 영역별로 적합한 에이전트 선택
       • Quality Agent (QA 페르소나)
       • Security Agent (보안 페르소나)
       • Performance Agent (성능 페르소나)
       • Architecture Agent (아키텍트 페르소나)

3. 병렬 실행
   └─> 여러 에이전트가 동시에 각자의 작업 수행

4. 결과 통합
   └─> 모든 결과를 종합하여 최종 리포트 생성


🔹 실제 사용 예시
----------------------------------------------------------------------

예시 1: 대규모 코드베이스 분석
  명령: /analyze --comprehensive
  결과: 자동으로 여러 Subagent가 병렬로 분석
        - Quality Agent: 코드 품질 검사
        - Security Agent: 보안 취약점 분석
        - Performance Agent: 성능 병목 지점 확인

예시 2: 파일별 작업 분산
  플래그: --delegate files
  효과: 50개 이상 파일을 여러 에이전트가 나눠서 처리
  성능: 40-70% 시간 단축

예시 3: 디렉토리별 작업 분산
  플래그: --delegate folders
  효과: 7개 이상 디렉토리를 병렬 처리
  성능: 최대 70% 효율 향상

│ 적합한 상황     │ 명확한 단일 작업     │ 대규모 복잡한 작업   │

4️⃣ 조합 사용 팁
======================================================================

🎯 최고의 효율을 위한 조합
----------------------------------------------------------------------

1. Skill + Subagent 자동 활성화
   예: /analyze --comprehensive
   → /analyze Skill 실행 + Subagent 자동 병렬 분석

2. 명시적 Subagent 제어
   예: /improve --delegate files --concurrency 10
   → 파일별 개선 작업을 10개 에이전트가 동시 처리

3. 복잡한 프로젝트 로딩
   예: /load @monorepo/ --delegate folders
   → 모노레포 전체를 폴더별로 병렬 로딩


🚀 성능 최적화 팁
----------------------------------------------------------------------
✓ 대규모 작업: --delegate auto 사용
✓ 빠른 탐색: Explore Agent 활용
✓ 병렬 처리: --concurrency 조정 (기본 7, 최대 15)
✓ 리소스 절약: 필요한 만큼만 에이전트 활성화

5️⃣ 실전 활용 예시 (한국여행 지도 프로젝트)
======================================================================

🗺️ 시나리오 1: 새로운 지역 여행 프로젝트 시작 (부산)
----------------------------------------------------------------------
현재 상황: 부산 여행 계획을 위한 인터랙티브 지도 제작

단계 1: 프로젝트 초기화
  명령: /korea-map-init --region busan --days 4 --theme modern-minimal

  실행 과정:
    → Architect Persona 자동 활성화
    → 디렉토리 구조 생성 (/, /data, /images)
    → index.html, script.js, styles.css 자동 생성
    → 부산 샘플 데이터 5개 추가
    → package.json 생성 (로컬 서버 스크립트 포함)

  결과:
    ✅ 완전히 작동하는 기본 프로젝트 생성 (30초 소요)
    ✅ 브라우저에서 바로 확인 가능


단계 2: 부산 관광지 데이터 대량 수집
  명령: /data-collector --type attractions --region busan --count 30 --auto-geocode

  (자동) Subagent 활성화:
    → GeoJSON Expert: 주소를 위도/경도로 자동 변환
    → Travel Data Curator: 데이터 검증 및 중복 제거
    → General-Purpose Agent: 한국관광공사 API 호출

  실행 과정:
    1. 한국관광공사 Tour API에서 부산 관광지 데이터 수집
    2. 주소 → 좌표 자동 변환 (카카오 API 사용)
    3. 데이터 정제: 이름 표준화 "해운대 (Haeundae Beach, 海云台)"
    4. JSON 스키마 검증
    5. data/korea-travel-data.json에 저장

  결과:
    ✅ 30개 관광지 데이터 자동 수집 및 저장 (2분 소요)
    ✅ 좌표, 주소, 설명, 가격 정보 모두 포함


단계 3: 맛집 정보 추가
  명령: /data-collector --type restaurants --keyword "부산 해운대 맛집" --count 15 --append

  (자동) Subagent 활성화:
    → Web Search Agent: 네이버/카카오 검색
    → Travel Data Curator: 기존 데이터와 병합, 중복 방지

  결과:
    ✅ 15개 맛집 정보 기존 데이터에 추가
    ✅ 중복 제거 완료


📊 시나리오 2: 기존 CSV 데이터를 지도 프로젝트로 변환
----------------------------------------------------------------------
현재 상황: "부산여행_브라운스위트부산역_맛집_관광지.csv" 파일 존재

명령: /implement "CSV 파일을 읽어서 JSON으로 변환하고 지도에 표시"

실행 과정:
  1. Analyze Agent → CSV 파일 구조 분석
  2. (자동) Subagent 활성화:
     → Data Processing Agent: CSV 파싱 및 변환
     → GeoJSON Expert: 주소 → 좌표 변환
     → Travel Data Curator: 데이터 검증

  3. Python 스크립트 자동 생성:
     ```python
     import csv, json

     def convert_csv_to_json(csv_file):
         with open(csv_file, 'r', encoding='utf-8') as f:
             reader = csv.DictReader(f)
             data = []
             for row in reader:
                 # 좌표 변환 로직
                 lat, lng = geocode(row['주소'])
                 data.append({
                     "name": row['이름'],
                     "latitude": lat,
                     "longitude": lng,
                     "type": row['타입']
                 })

         with open('data/korea-travel-data.json', 'w') as f:
             json.dump(data, f, ensure_ascii=False, indent=2)
     ```

  4. 스크립트 실행 및 검증

결과:
  ✅ CSV → JSON 자동 변환 완료
  ✅ 지도에 마커로 표시됨
  ✅ 데이터 품질 검증 완료


🎨 시나리오 3: 지도 디자인 개선 및 성능 최적화
----------------------------------------------------------------------
명령: /improve --focus performance --delegate files

(자동) Subagent 활성화:

  → UI Designer: 한국 전통색 적용
  → Leaflet Specialist: 마커 클러스터링 구현
  → Frontend Agent: 반응형 레이아웃 개선

  4. Frontend Agent:

     - 터치 제스처 지원

결과:
  ✅ 로딩 속도 50% 향상 (3.2초 → 1.6초)

  ✅ 모바일 사용성 대폭 개선
  ✅ 한국 전통 디자인 적용 완료


🔍 시나리오 4: 데이터 품질 검증 및 중복 제거
----------------------------------------------------------------------
명령: /analyze --focus quality data/korea-travel-data.json --delegate auto

(자동) Subagent 활성화:
  → Travel Data Curator: 데이터 품질 분석 전문
  → Explore Agent: 중복 데이터 검색

실행 과정:
  1. JSON 스키마 검증
     - 필수 필드 확인 (name, latitude, longitude, type)
     - 좌표 범위 검증 (한국 영역: 위도 33-38.5, 경도 124-132)

  2. 중복 탐지
     - 이름 유사도 분석 (Levenshtein distance)
